How to build myBlog

hexo

  1. 安装hexo
    npm install hexo-cli -g

  2. 创建一个博客文件夹
    在文件夹中执行 hexo init

  3. 在文件夹中直接执行 hexo s 在本地测试网页效果

新建文章

hexo new “文章名称”

这句命令执行完毕之后，会在source/_posts/ 中生成以文章名为文件名的markdown文件

编辑文章

找到对应的markdown文件，编辑内容即可

生成html

hexo g

执行这条命令之后，会在public目录下生成所有的静态页面，只要将该目录下所有的文件进行发布，发布到github上，就可以通过github的静态页面托管服务来访问页面了

如何使用hexo d 来发布网站呢

  1. 安装 hexo-deployer-git
    npm install hexo-deployer-git –save

  2. 配置网站根目录下面的_config.yml
    配置最下面的deloy节点
    deploy:

    • type: git
      repo: github静态页面托管项目的地址

2.1 在public目录下执行 git init 并进行一次提交

  1. 按照正常流程创建博客， 编辑博客

  2. 执行hexo g

  3. 在 public 目录下去执行 hexo d

关于extend的使用方法

```bash

    /*jQuery的extend
     * 如果传入多个参数，那么把后面的对象混入给第一个成员
     * 如果传入1个参数，谁调用就是给谁copy
     * 
     */
    var a = {age :18};
    var b = {name:'四哥'};
    var c = {hobby:'逛吃'};
    $.extend(a,b,c);//就是把b,c都混入给a 相当于copy
    console.log(a);//Object {age: 18, name: "sige", hobby: "chi"}
    //给$添加静态成员
    $.extend(a)//把a混入$
    console.log($);
    $.fn.extend({
        con:function(msg){
            console.log(msg)
        }
    })
    $().con('打印试试');

    function extend(o1,o2){
        //混入只混入自己不混入继承过来的，这边做一个判断，判断是不是自己的hasOwnProperty
        for(var k in o2){
            if(o2.hasOwnProperty(k)){
                   o1[k] = o2[k];
            }

        }
    }
    //函数式子
    function extend1(obj){
        for(var key in obj){
            if(obj.hasOwnProperty(key)){
                this[key] = obj[key];
            }
        }
    }
    //比如
    var o = {
        a:1,
        ext:extend1
    }
    o.ext({
        val:'我是混入的值'
    })
    console.log(o);
